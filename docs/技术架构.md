# 技术架构文档

## 一、系统架构概览

本系统采用"内核态采集 + 用户态分析"的架构设计，充分利用eBPF技术的非侵入式特性。

```
┌─────────────────────────────────────────────────────────────────┐
```

## 二、核心模块说明

### 2.1 eBPF 内核态程序 (src/bpf/main.bpf.c)

#### Tracepoints 监控

| Tracepoint | 功能 | 数据获取 |
|------------|------|---------|
| sys_enter_execve | 进程执行监控 | 命令行参数 |
| sys_exit_clone | 进程创建监控 | 子进程PID |
| sys_enter_openat | 文件打开监控 | 文件路径、标志位 |
| sys_enter_unlinkat | 文件删除监控 | 文件路径 |
| sys_exit_connect | 网络连接监控 | 连接结果 |
| sys_exit_accept4 | 网络接受监控 | 连接结果 |

#### Kprobes 监控

| Kprobe | 功能 | 数据获取 |
|--------|------|---------|
| tcp_connect | TCP连接详情 | 源/目的IP、端口 |

#### Uprobes 监控

| Uprobe | 功能 | 数据获取 |
|--------|------|---------|
| SSL_read | HTTPS响应解密 | 明文响应数据 |
| SSL_write | HTTPS请求解密 | 明文请求数据 |

## 三、数据流设计

### 3.1 事件数据结构

```c
// 进程事件
struct process_event_data {
    struct process_context ctx;  // 进程上下文
    __u32 event_type;            // 事件类型
    __u32 target_pid;            // 目标PID
    char args[MAX_ARGS_LEN];     // 命令参数
    __s32 ret_val;               // 返回值
};

// 文件事件
struct file_event_data {
    struct process_context ctx;
    __u32 event_type;
    char path[MAX_PATH_LEN];     // 文件路径
    __u32 flags;                 // 打开标志
    __u32 mode;                  // 模式
    __s32 ret_val;
};

// 网络事件
struct network_event_data {
    struct process_context ctx;
    __u32 event_type;
    __u32 family;                // 地址族
    __u16 local_port;
    __u16 remote_port;
    __u32 local_ip;
    __u32 remote_ip;
    __s32 ret_val;
};

// SSL事件
struct ssl_event_data {
    struct process_context ctx;
    __u32 event_type;
    __u64 fd;
    __u32 data_len;
    char data[MAX_DATA_LEN];     // 明文数据
};
```

### 3.2 数据流路径

```
1. 系统调用发生
   ↓
2. eBPF程序捕获 (Tracepoint/Kprobe/Uprobe)
   ↓
3. 构造事件数据结构
   ↓
4. 写入Ring Buffer
   ↓
5. 用户态程序读取
   ↓
6. 事件分类处理
   ↓
7. 异常检测分析
   ↓
8. 告警输出
```

## 四、关键技术实现

### 4.1 BTF CO-RE (Compile Once, Run Everywhere)

```c
// 使用 BPF_CORE_READ 实现内核版本兼容
struct task_struct *parent = BPF_CORE_READ(task, real_parent);
ctx->ppid = BPF_CORE_READ(parent, tgid);
```

### 4.2 Ring Buffer 高效传输

```c
// 内核态写入
event = bpf_ringbuf_reserve(&process_events, sizeof(*event), 0);
// 填充数据...
bpf_ringbuf_submit(event, 0);

// 用户态读取
rb_process = ring_buffer__new(bpf_map__fd(map), process_event_cb, NULL, NULL);
ring_buffer__poll(rb_process, 100);
```

### 4.3 Uprobe HTTPS 解密

```c
// Hook SSL_read
SEC("uprobe")
int uprobe_ssl_read(struct pt_regs *ctx)
{
    void *buf = (void *)PT_REGS_PARM2(ctx);
    int num = (int)PT_REGS_PARM3(ctx);
    // 读取明文数据
    bpf_probe_read_user(event->data, num, buf);
}
```

## 五、性能优化策略

### 5.1 已实现

1. **Ring Buffer**: 比 Perf Buffer 更高效
2. **Per-CPU Array**: 避免锁竞争
3. **BTF CO-RE**: 避免内核版本依赖

### 5.2 待实现

1. **事件过滤**: 在内核态过滤无关事件
2. **批量处理**: 批量读取事件减少系统调用
3. **采样策略**: 高负载时采样处理
