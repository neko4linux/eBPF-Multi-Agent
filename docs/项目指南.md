# 项目指南 - 面向参赛队员

## 一、赛题内容详细解释

### 1.1 赛题背景

随着大语言模型技术的发展，AI Agent（智能体）正在从被动应答向自主规划、协同执行复杂任务演进。例如 Claude Code、Gemini CLI 等端侧智能体可以多智能体协作完成编程、运维等任务。

**核心问题**：智能体的高度自主性带来潜在风险：
- 模型幻觉引发的逻辑死循环 → 计算资源浪费
- 恶意提示词注入 → 诱导执行高危操作
- 多智能体并发场景 → 难以精准定位异常根因

**现有工具局限**：
- 应用层工具数据预处理复杂、检测延迟大
- 缺乏操作系统层面的全局视图
- 难以建立模型输出与系统操作的映射关系

### 1.2 赛题要求详解

#### 任务一：多层级数据捕获

**基础要求（必须完成）**：
| 监控类型 | 系统调用 | 目的 |
|---------|---------|------|
| 进程管理 | execve, fork | 监控进程创建和程序执行 |
| 文件操作 | open, unlink | 监控文件读写和删除 |
| 网络通信 | connect, accept | 提取连接的IP与端口 |

**进阶要求（加分项）**：
- 利用 uprobes 技术 Hook OpenSSL 等加密库
- 非侵入式截获 HTTPS 明文通信
- 获取 Prompt/Response 内容

#### 任务二：异常监测

**基础要求**：
| 异常类型 | 检测方法 |
|---------|---------|
| 逻辑死循环 | 高频API调用 + 重复Prompt + 无意义文件读写 |
| 资源浪费 | 综合判定资源使用异常 |
| 非预期Shell启动 | 检测 /bin/sh, /bin/bash 等启动 |
| 敏感文件越权访问 | 监控 /etc/passwd, /etc/shadow 等 |
| 工作区外文件删除 | 检测超出工作区范围的删除操作 |

**进阶要求**：
- 多智能体协作或竞争场景下的异常检测

#### 任务三：告警输出与归因定位

**基础要求**：
- 输出包含时间戳、进程PID/TID、操作对象的日志

**进阶要求**：
- 突破"语义鸿沟"
- 建立跨层数据关联策略
- 实现 Prompt 与底层系统异常的精确因果匹配

## 二、赛题意义

### 2.1 技术意义

1. **操作系统与AI融合**：探索操作系统层面支持AI应用的新方向
2. **eBPF技术应用**：推动eBPF在可观测性、安全领域的发展
3. **非侵入式监测**：无需修改内核或应用程序的监测方案

### 2.2 实际应用价值

1. **AI安全**：防止AI Agent被恶意利用
2. **资源保护**：避免AI Agent异常消耗计算资源
3. **审计追溯**：为AI行为提供可追溯的日志

根据赛题要求，以下部分**禁止修改**：

## 八、技术优化方案

### 8.1 已使用的技术优化

| 技术 | 优化目标 | 效果 |
|-----|---------|------|
| Ring Buffer | 事件传输效率 | 高效、低延迟 |
| BTF CO-RE | 内核版本兼容 | 无需为每个内核版本编译 |
| Per-CPU Array | 减少锁竞争 | 提高并发性能 |
| Tracepoints | 稳定的内核接口 | 比kprobe更稳定 |

### 8.2 后续优化方向

1. **性能优化**：
   - 事件过滤下推到内核态
   - 批量事件处理
   - 内存池优化

2. **功能优化**：
   - 支持更多加密库（GnuTLS, mbedTLS）
   - 规则引擎可配置化
   - 机器学习异常检测

3. **工程优化**：
   - 配置文件支持
   - 日志分级
   - 监控面板

## 九、其他重要说明

### 9.1 项目架构

```
┌─────────────────────────────────────────┐
│           用户态 (User Space)            │
│  ┌─────────┐ ┌─────────┐ ┌───────────┐  │
│  │主程序   │ │异常检测 │ │告警管理   │  │
│  └────┬────┘ └────┬────┘ └─────┬─────┘  │
│       └───────────┴────────────┘        │
│              Ring Buffer                 │
└───────────────────┬─────────────────────┘
                    │
┌───────────────────┴─────────────────────┐
│           内核态 (Kernel Space)          │
│  ┌─────────────────────────────────────┐│
│  │        eBPF 程序                     ││
│  │  Tracepoints | Kprobes | Uprobes    ││
│  └─────────────────────────────────────┘│
└─────────────────────────────────────────┘
```

### 9.2 快速开始

```bash
# 编译
make all

# 运行（需要root权限）
sudo ./build/agent-monitor -f build/main.bpf.o

# 测试
sudo ./scripts/test.sh
```
